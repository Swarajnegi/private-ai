# =============================================================================
# JARVIS — Python dependencies (production + learning)
#
# Covers everything jarvis_core/, scripts/, and js-learning/ import.
# Pinned to versions verified working on work-laptop (Linux/WSL Ubuntu 22.04,
# Python 3.10.12) on 2026-04-30. Cross-laptop tested: same pins should
# resolve cleanly on personal-laptop (Windows + WSL Ubuntu).
#
# Setup on a fresh machine:
#     python3 -m venv .venv
#     # If `python3 -m venv` errors with "ensurepip not available", install:
#     #   sudo apt install python3.10-venv     (Ubuntu/Debian)
#     # Or use virtualenv as a no-sudo fallback:
#     #   pip3 install --user virtualenv && python3 -m virtualenv .venv
#     source .venv/bin/activate
#     pip install -r requirements.txt
#
# Then verify:
#     python -c "from jarvis_core.memory.store import JarvisMemoryStore; print('OK')"
#
# Why CPU-only torch:
#     JARVIS architecture (per .agent/rules/JARVIS_ENDGAME.md) runs embeddings
#     + ChromaDB on the laptop CPU and offloads LLMs to cloud APIs. CUDA torch
#     would 4x the download (~750 MB vs ~190 MB) for zero runtime benefit on
#     this hardware profile. If you later add a GPU, swap the torch line to
#     `torch==<version>` and drop the --extra-index-url.
# =============================================================================
